Help

Network Penetration Testing Tool is a website project designed to help Small and Medium Enterprise (SME) administrators test the security of their network devices. Users can input an IP range or specific IP address to scan network devices and identify open ports. The tool uses nmap for scanning and service version detection, as well as script scanning for various services such as MySQL, SMB, SSH, and HTTP.

In addition to scanning, users can set up a whitelist to identify devices that are not on the list and view a summary of the scan results. The summary provides valuable information such as the total number of open ports found, the most common open ports, the percentage of ports used for web traffic, the least common open port, the percentage of whitelisted devices, and the number of non-whitelisted devices that may pose a security risk. This tool aims to assist SME administrators in identifying potential vulnerabilities and improving their network security.

To start scanning the network, follow these steps:

1. On the "Network Penetration Testing Tool" website click the "Scanner" tab, input the IP address or IP range you want to scan (e.g., 192.168.10.0/24 or 192.168.10.1-5) in the provided field.
2. Click the "Scan" button and wait for the scanning results to be displayed. The results will show which devices are not on the whitelist.
3. If you want to scan the device services version, click "More" and then click "Scan services version". Wait for the scanning services results to be displayed, which will show the services information.
4. If you want to use script scanning, select the desired script from the list and click the script to run.
5. Review the script results to analyze the network's security and identify any potential vulnerabilities.

A whitelist is a list of approved devices that are allowed to access a network or system. In the context of the "Network Penetration Testing Tool" website, the whitelist is used to identify devices that are authorized to be on the network. Devices not on the whitelist may indicate a security risk, as they could be unauthorized or potentially malicious.

There are two methods to add a device to the whitelist:

Type 1: Add the device manually through the website input box.
1. Go to the whitelist page and locate the "Add MAC Address to whitelist" section on the left-hand side.
2. Enter the device's MAC address and an optional remark.
3. Click the "Add to whitelist" button to add the device to the whitelist.

Type 2: Upload a CSV file with the device information.
1. Go to the whitelist page and locate the "Upload a csv file to whitelist (.csv)" section on the right-hand side.
2. Download the whitelist template to ensure your CSV file has the correct format.
3. Select your CSV file and click the "Upload the file" button to add the devices listed in the file to the whitelist.

In the whitelist table, you can remove individual devices from the whitelist or remove all devices by clicking the "Remove" or "Remove All Whitelist" buttons, respectively. You can also download the entire whitelist by clicking the "Download All Whitelist" button.

The summary page is a section of the "Network Penetration Testing Tool" website that provides an overview of the scanning results. It displays important information such as the total number of open ports found in the scanned network, the most common open ports, the percentage of open ports used for web traffic, the least common open port, the percentage of whitelisted devices, and the number of devices not in the whitelist that may pose a security risk. The summary page helps SME administrators to quickly assess the security of their network and take necessary actions, such as adding devices to the whitelist.

Supports private IPv4 addresses in the following formats:

- 10.0.0.0/8 IP addresses: 10.0.0.0 – 10.255.255.255
- 172.16.0.0/12 IP addresses: 172.16.0.0 – 172.31.255.255
- 192.168.0.0/16 IP addresses: 192.168.0.0 – 192.168.255.255

It also supports optional CIDR or hyphen notation at the end of the IP address string (e.g., /24 or -16).
